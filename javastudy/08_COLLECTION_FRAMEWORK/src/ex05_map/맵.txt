Map 인터페이스

1. 하나의 데이터를 '키(key)-값(value)'으로 구성하는 자료구조이다.
2. '키(key)'를 '변수', '값(value)'을 '변수에 저장된 값'으로 생각하면 된다.
3. 특징
    1) 키(key)는 중복이 불가능하다. 값(value)을 얻기 위해 필요한 데이터이다.
    2) 값(value)은 중복이 가능하다. 실제로 저장하고자 하는 데이터이다.
4. HashMap 클래스
    1) 해시 기반의 Map 처리 클래스이다.
    2) 해시(Hash) : 특정 인스턴스가 가지는 값. 빠른 검색을 위해서 사용.
5. HashMap 클래스의 키(key)의 중복 요소 여부 판단
    1) 2가지 메소드를 사용한다.
    2) hashCode() 메소드를 호출해서 해시(Hash)값을 확인한다.
        - 동일한 해시 : 중복된 요소일 가능성 있음. 추가로 equals() 비교함.
        - 다른 해시 : 중복된 요소가 아님 확정. 더 이상 비교 안 함.
    3) equals() 메소드를 호출해서 동등 비교를 수행함.
    4) 실무에서는 hashCode(), equals() 메소드를 동시에 자동 생성함.
